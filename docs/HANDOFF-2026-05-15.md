## Agent Browser

Respuesta arquitectonica: si, Agent Browser tiene sentido con WSL2. El bloqueo era Windows-native; en WSL2 se ejecuta como Linux y evita la mayoria de problemas de paths/sockets/Git Bash.

Decision pragmatica:

- Mantener Playwright como fallback actual porque ya esta implementado y probado.
- Agregar Agent Browser despues de que WSL2 + lifecycle + supervisor esten estables.
- Usarlo como etapa 2 preferida para browser automation token-efficient, con perfil dedicado y sin tocar Chrome personal.

Referencia primaria: `https://github.com/vercel-labs/agent-browser`.
